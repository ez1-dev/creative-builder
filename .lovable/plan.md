## Objetivo

Criar a tabela `public.producao_recurso_unidade` no Lovable Cloud para mapear recursos do ERP (codcre) a unidades de negócio, especialmente para recursos sem centro de custo (012, 013, 1100, 1200, 3132, 4000, 90, 95).

## Migração SQL

Será executada via ferramenta de migração:

```sql
CREATE TABLE IF NOT EXISTS public.producao_recurso_unidade (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  codemp INTEGER NOT NULL,
  codcre TEXT NOT NULL,
  descre TEXT,
  unidade_negocio TEXT NOT NULL,
  tipo_recurso TEXT,
  codccu_sugerido TEXT,
  considera_carga BOOLEAN NOT NULL DEFAULT TRUE,
  ativo BOOLEAN NOT NULL DEFAULT TRUE,
  obs TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (codemp, codcre)
);

CREATE INDEX IF NOT EXISTS idx_producao_recurso_unidade_codemp ON public.producao_recurso_unidade (codemp);
CREATE INDEX IF NOT EXISTS idx_producao_recurso_unidade_codcre ON public.producao_recurso_unidade (codcre);
CREATE INDEX IF NOT EXISTS idx_producao_recurso_unidade_unidade ON public.producao_recurso_unidade (unidade_negocio);
CREATE INDEX IF NOT EXISTS idx_producao_recurso_unidade_ativo ON public.producao_recurso_unidade (ativo);
```

## RLS (segurança)

A tabela precisa de RLS — proposta padrão do projeto:

- **Leitura**: qualquer usuário autenticado (`authenticated`) pode ler — necessário para o ETL/relatórios de produção consumirem o mapeamento.
- **Escrita (INSERT/UPDATE/DELETE)**: apenas administradores (`is_admin(auth.uid())`), já que é tabela de cadastro mestre operacional.

Trigger `update_updated_at_column()` (já existente no projeto) será conectado para manter `updated_at`.

## Fora do escopo

- Não vou criar tela de CRUD nem alterar nenhum código frontend nesta etapa — só a estrutura no banco. Telas/uso podem vir em pedido separado.
